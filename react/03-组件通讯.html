组件通讯 #

组件是独立作用域，默认情况下，只能使用组件自己的数据。在组件化过程中，我们将一个完整的功能 拆分成多个组件，以更好的完成整个应用的功能。而在这个过程中，多个组件之间不可避免的要共享某些数据 。为了实现这些功能，就需要打破组件的独立封闭性，让其与外界沟通。这个过程就是组件通讯。

props #

  • props 的作用：接收传递给组件的数据
  • props 的值：是标签属性组成对象。

使用步骤：

  • 传递数据：给组件标签添加属性。 -- 与 vue 类似
  • 接收数据：函数组件通过参数 props 接收数据，类组件通过 this.props 接收数据

props 的特点 #

  • 可以给组件传递任意类型的数据，包括虚拟 DOM 结构
    • 字符串、数字、
    • 数组、对象、
    • 💥 函数、💥JSX
  • 💥props 是只读的，不允许修改 props 的数据 - 单向数据流
    • 💥React 中 props 绕不过引用地址。

总结：

  1. props 是对象
  2. props 可以是任意值

props 深入-children 属性 #

children 属性：表示该组件的子节点，只要组件有子节点，props 就有该属性

children 属性与普通的 props 一样，值可以是任意值（文本、React 元素、组件，甚至是函数）

js
function Hello(props) {
  return <div>该组件的子节点：{props.children}</div>;
}

组件通讯三种方式 #

  • 父传子
  • 子传父

父传子 #

  1. 父组件提供要传递的 state 数据
  2. 给子组件标签添加属性，值为 state 中的数据
  3. 子组件中通过 props 接收父组件中传递的数据

静态结构准备

jsx
import React, { useState } from "react";
export default function App() {
  const [money, setMoney] = useState(99);

  const handleMakeMoney = () => {
    setMoney(1000 + money);
  };

  return (
    <div>
      <button onClick={handleMakeMoney}>爸爸开始赚钱了</button>
      <Child></Child>
    </div>
  );
}

function Child() {
  return (
    <div>
      <h1>爸爸给我钱了： </h1>
    </div>
  );
}

子传父 #

思路：利用回调函数，父组件提供回调，子组件调用，将要传递的数据作为回调函数的参数。

  1. 父组件提供一个回调函数（用于接收数据）
  2. 将该函数作为标签属性的值，传递给子组件
  3. 子组件通过 props.语法找到函数，再 调用回调函数
  4. 将子组件的数据作为参数传递给回调函数

使用步骤：

父组件提供函数，并且通过标签属性，传递给子组件

diff
import React, { useState } from 'react';
export default function App() {
  const [money, setMoney] = useState(99);

  const handleMakeMoney = () => {
    setMoney(1000 + this.state.money);
  };

// 1. 父组件提供一个回调函数
+  const handleCostMoney = () => {
+    setMoney(money - 100);
+  };

  return (
    <div>
      <button onClick={handleMakeMoney}>爸爸开始赚钱了</button>
      <Child
+      	// 2. 将该函数作为标签属性的值，传递给子组件
+      	handleCostMoney={handleCostMoney}
			/>
    </div>
  );
}

+function Child({ handleCostMoney }) {
  return (
    <div>
      <h1>爸爸给我钱了： </h1>
      <button
+      	onClick={handleCostMoney}
			>吞金兽要花钱</button>
    </div>
  );
}

兄弟组件 #

  • 将共享状态提升到最近的公共父组件中，由公共父组件管理这个状态
  • 思想：状态提升
  • 公共父组件职责：
    • 提供共享状态
    • 提供操作共享状态的方法
  • 要通讯的子组件只需通过 props 接收状态或操作状态的方法

跨组件通信-Context #

基本概念 #

思考：App 组件要传递数据给 Child 组件，该如何处理？

处理方式：使用 props 一层层组件往下传递（繁琐）

更好的姿势：使用 Context

作用：跨组件传递数据（比如：主题、语言等）

App.js 直接复制如下静态结构

jsx
/* 
  目标： 使用Context来跨组价通信，让App组件-直接传数据给SonSon组件
*/

export default function App() {
  return (
    <div>
      <h1>父组件</h1>
      <Son></Son>
    </div>
  );
}

function Son() {
  return (
    <div>
      <h2> 儿子</h2>
      <SonSon></SonSon>
    </div>
  );
}

function SonSon() {
  return (
    <div>
      <h2> 孙子</h2>
    </div>
  );
}

实现步骤 #

  • 调用 React. createContext() 创建 Provider（提供数据） 和 Consumer（消费数据） 两个组件。
js
const { Provider, Consumer } = React.createContext();
1
  • 使用 Provider 组件作为父节点。
js
<Provider>
  <div className="App">
    <Child1 />
  </div>
</Provider>
1
2
3
4
5
  • 设置 value 属性，表示要传递的数据。
js
<Provider value="pink">
1
  • 调用 Consumer 组件接收数据。
js
<Consumer>{(data) => <span>data参数表示接收到的数据 -- {data}</span>}</Consumer>
1

总结：

  1. 如果两个组件是远方亲戚（比如，嵌套多层）可以使用 Context 实现组件通讯
  2. Context 提供了两个组件：Provider 和 Consumer。
  3. createContext()可以调用多次，每次解构出来的 Provider 和 Consumer 要成对使用。
  4. Provider 组件：用来存入数据
  5. Consumer 组件：用来取出数据
