defineProps的TS写法 ​

  1. defineProps 的基本使用：
ts
const props = defineProps({
  money: {
    type: Number,
    required: true
  },
  car: {
    type: String,
    required: false,
    default: '宝马车'
  }
})
console.log(props.money) // number
console.log(props.car) // string | undefined

ref的TS写法 ​

ref() 会隐式的依据数据推导类型

  1. 如果是简单类型，推荐使用类型推导：
ts
// const money = ref<number>(10)

const money = ref(10)
1
2
3
  1. 如果是复杂类型，推荐指定泛型：
ts
type Todo = {
  id: number
  name: string
  done: boolean
}
const list = ref<Todo[]>([])

setTimeout(() => {
  list.value = [
    { id: 1, name: '吃饭', done: false },
    { id: 2, name: '睡觉', done: true }
  ]
}, 1000)

  2. 复杂数据类型，完整写法

reactive的TS写法 ​

reactive() 也会隐式的依据数据推导类型

  1. 默认值属性是固定的，推荐使用类型推导：
ts
// 推导得到的类型：{ title: string }
const book = reactive({ title: 'Vue3 在线医疗' })
1
2
  1. 根据默认值推导不出我们需要的类型，推荐使用接口或者类型别名给变量指定类型：
ts
// 我们想要的类型：{ title: string, year?: number }
type Book = {
  title: string
  year?: number
}
const book: Book = reactive({ title: 'Vue3 在线医疗' })
book.year = 2022

Template Ref与TS ​

模板 ref 需要通过一个显式指定的泛型参数，建议默认值 null

vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'

const el = ref<HTMLInputElement | null>(null)

onMounted(() => {
  el.value?.focus()
})
</script>

<template>
  <input ref="el" />
</template>

非空断言 ​

处理类型可能是 null 或 undefined 的值，下面的属性或函数的访问赋值：

  1. 可选链
vue
<script setup lang="ts">
import { onMounted, ref } from 'vue';


const input = ref< HTMLInputElement | null >(null)

onMounted(()=>{
  // 可选链：只能访问，不能赋值
  // input.value?.value = "123" ❌
  console.log(input.value?.value);
})

</script>

<template>
  <div>App组件</div>
  <input type="text" ref="input" value="abc">
</template>
